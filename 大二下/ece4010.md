## 搜索

Frontier（前沿）：某种用于暂存node的容器

算法框架：

* 有无已探索集

  1. 通用搜索**（没有已探索集）**

     * 前沿最初包含初始节点
     * 重复：
       1. 若前沿中没有节点则无解
       2. 移除第一个节点
       3. 若包含最终节点则返回最终节点
       4. 将移除的节点的子节点加入前沿中

     问题：不包含已探索集，所以可能发生死循环

  2. 图搜索**（有已探索集）**

     * 前沿最初包含初始节点，已探索集为空集

     * 重复：
       1. 若前沿中没有节点则无解
       
       2. 移除第一个节点
       
       3. 若包含最终节点则返回最终节点
       
       4. 将移除的节点加已探索集
       
       5. 将移除的节点的子节点加入前沿中
       
          

* 有无启发函数（地图信息）

  1. 无信息搜索**（没有启发函数）**

     1. 深度优先搜索DFS

        **用堆栈作为前沿**的图搜索

     2. 广度优先搜索BFS

        **用队列作为前沿**的图搜索

  2. 有信息搜索**（有启发函数）**

     启发函数其实就是让前沿变成了某种优先队列

     1. 贪婪最佳优先算法

        启发算法：只关心当前节点到目标节点的估算距离（曼哈顿距离），选择估算距离最小的节点

        曼哈顿距离：横向差值+纵向差值

     2. A*算法

        启发算法：选择路径损耗+估算距离最小的节点

## 对抗

* 新函数

  PLAYER(s)：返回哪方走下一步

  TERMINAL(s)：判断游戏是否结束

  UTILITY(s)：游戏结束时的分值

1. MINIMAX 算法

   从当前状态开始通过交替递归预测之后所有的回合。自己的回合选择自己得分最高的动作，对手的回合选择自己得分最低的动作

   * MAX：选择分值最高的走法（自己）
   * MIN：选择分值最低的走法（对手）

   1. 代码示例：获得最小分值的动作

      ```python
      def MIN-VALUE(state):
      	if TERMINAL(state):
      		return UTILITY(state)
          
      	v = +∞
      	for action in ACTIONS(state):
      		v = MIN(v, MAX-VALUE(RESULT(state, action)))
      	return v
      ```
      
      v是当前遍历过的最小值（初始为无穷大）
      
      
      

2. ab剪枝

   **一个最大值节点下面的最小值节点，如果发现自己的值已经小于最大值当前的取值了，就立刻舍弃自身**

   代码示例：最大值节点

   ```python
   def MAX-VALUE (state, a, b):
   	if TERMINAL(state):
           return UTILITY(state)
       
       v = -∞
       for action in ACTIONS(state):
           v_temp = MAX-VALUE(RESULT(state, action), a, b)
           v = MAX(v, v_temp)
           a = MAX(a, v_temp)
           if a >= b:
               break
       return v
   ```



## 逻辑

1. 命题

   命题就是某种论述，用大写字母表示

   命题之间可以通过逻辑符号连接，表示命题之间的逻辑关系

   逻辑符号：

   * 非 $\neg$ 
   * 与 $\and$
   * 或 $\or$
   * 推导 $\rightarrow$ (仅当P为真，Q为假的时候，$P \rightarrow Q$才为假)
   * 互推 $\leftrightarrow$ (仅当P，Q均为真或假的时候，$P \leftrightarrow Q$才为真)

2. 模型

   模型类似于真值表，是给所有命题赋值的一种可能情况

   知识库（KB）：agent已知的一系列语句

   Entailment：如果在任何模型中，只要$\alpha$为真，$\beta$就一定为真，则称$\alpha \models \beta$（这是a蕴含在b中）

   * 推导更像一种函数，接收两个命题并返回一个结果
   * 蕴含是一系列陈述（所有的模型）与一个具体陈述（ab关系）之间的关系

   推理：从旧有的陈述（KB）中得出新陈述的过程

   

   模型检查：判断KB是否蕴含在某个语句中

   * 首先枚举出所有可能的模型





## 概率

1. 贝叶斯网络：$P(X_1,\dots,X_N)=\prod_{n=1}^NP(X_n\mid X_1,\dots,X_{n-1})=\prod_{n=1}^NP(X_n\mid \text {Parents}(X_n))$

   简单来说，就是这个公式嵌套：$P(AB)=P(A\mid B)\cdot P(B)$ （然后继续拆B）

2. 概率计算

   1. 枚举推理：硬算。把所有隐藏变量都枚举出来，再按照比值算出目标事件的概率

   2. 近似推理：依靠采样进行计算。采样就是直接实验然后统计次数。

      1. 先验采样：硬抽，从贝叶斯网络的顶端开始，抽中第一层就抽下一层以此类推，最后统计样本

      2. 拒绝采样：一样硬抽，但是抽出来的样本如果不符合已知条件就立刻舍弃

      3. 似然加权：硬抽，但是抽出来的样本如果是已知条件就跳过并在这个样本的权重中乘上当前背景下发生已知条件的概率

      4. 吉布斯采样：每次根据其他变量的情况抽取并更新一个非证据变量，完整一轮会把所有非证据变量都更新一遍，每一轮产生一个新样本
