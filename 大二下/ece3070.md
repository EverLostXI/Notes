## System Verilog 快速上手

1. module块

   用来定义你的元件

   ```verilog
   module example (
       // 在开头的括号中，可以定义这个元件的输入和输出
       input logic a, b,
       output logic f
   );
       // 还可以创建元件的内部信号并为它赋“值”，类似变量
       logic na;
       assign na = ~a;
       
       // 可以实例化逻辑门，并且把你定义的信号接入其中。第一位是输出，后面可以有不定位数的输入
       and g0 (f, na, b);
       
   endmodule
   ```

   

2. 常用语法

   * 多位宽信号

   ```verilog
   // RT设计中需要多位宽的数据传输，可以在定义元件输入输出的时候设置
   module example (
       // 可以用parameter语法设置参数，方便修改位宽
       #(parameter W = 3)
       // 然后直接给输入输出信号设置位宽
       input logic [W-1: 0] D,
       output logic [W-1:0] Q
   );
   ```

   

   * always逻辑

   ```verilog
   // always语法检测指定信号的特征，在检测到时会立刻跳转到这个部分的代码块运行
   // 后面的_ff和_comb决定了代码块中的内容将会以时序逻辑还是组合逻辑运行
   // 如果是组合逻辑，则每一行代码会立即执行
   // 如果是时序逻辑，则程序会先保留一份当前状态的快照，在代码全部执行完成后同时进行赋值
   always_ff@(posedge clk or negedge clk) begin
       if (!clk) begin
           q <= b;
       end else begin
           q <= 0;
       end
       
   end
   ```

## 时序延迟

<small>虽然很烦，但这个确实有点重要</small>

1. 根本思路

   T~cyc~ >= T~cq~ + T~comb~ + T~setup~（最长不能超过一个周期长）

   T~cq~ + T~comb~ >= T~hold~（为了运算顺利完成寄存器得在计算开始后保持一段时间）

2. 术语和意义

   讨论组合逻辑电路时，一般说的是两个寄存器之间的组合电路部分。

   这个部分有总输入PI和总输出PO

   

   * critical path（最长路径）：从PI到PO的最长延迟，**决定计算能不能在一个周期内完成**
   * short path（最短路径）：从PI到PO的最短延迟，**防止有些信号跑太快干扰还在保持输出的寄存器**

   根据这两个路径的作用，它们对应的延迟分别叫

   * 传播延迟
   * 污染延迟

3. 分析和计算

   从PO往回一个门一个门地遍历，就可以找出所有可能路径，再计算每条路径的门延迟之和即可

   * 传播延迟：路上所有门的传播延迟之和，找所有路径中的最大值
   * 污染延迟：路上所有门的污染延迟之和，找所有路径中的最小值

   

   但是需要注意：有些路径虽然物理上连接了，实际上信号有可能永远不经过它。这种“**伪路径**”不应该纳入计算

   <img src="image\3070\屏幕截图 2026-04-23 020740.png" alt="屏幕截图 2026-04-23 020740" style="zoom:67%;" />

   这些是触发器的一些时序性质：在CLK到来前后，其输入必须稳定；在CLK后一段时间内，其输出不稳定

   * CLK到来前

     T~su~设置时间：输入保持稳定的时间

   * CLK到来后

     T~h~维持时间：输入继续保持稳定的时间

     T~clk2Q~传播时间：输出达到稳定所需的时间

   













## 数据通路









工业级设计的最小单元不再是逻辑门，而是已经搭好，经过检验的基础元件。

* 硬件领域的RT设计规范，等同于软件领域的封装。

FSM-D架构（又称硬件线程）的设计，将电路分为两个主要部分：

1. 数据通路（Datapath）：用来处理数据的部分。由基础元件组成，如算术单元、寄存器
2. FSM：通过发送指令，控制数据通路的部分。是一个有限状态机。

* 数据通路中每条线可以包含多位数据
* 一个RT过程（寄存器$\rightarrow$组合逻辑$\rightarrow$寄存器）应该在一个时钟刻内完成
