2. 主定理

   如果整个算法的复杂度为形如：$T(n)\le a\cdot T(n/b)+O(n^d)$ 的式子

   则有：
   $$
   T(n) = \begin{cases} 
   O(n^d \log n) & \text{if } a = b^d \\ 
   O(n^d) & \text{if } a < b^d \\ 
   O(n^{\log_b(a)}) & \text{if } a > b^d 
   \end{cases}
   $$
   a：每次拆分出的子问题数量

   n / b：每个子问题的大小

   O(n^d^)：拆分这些问题所需要的固定步骤

5. 哈希

   1. 全域哈希的数学

      冲突事件 $h(x)=h(y)$ 可表示为所有位置 $a$ 上两个哈希值同时等于 $a$ 的并集：

      $\Pr[h(x)=h(y)] = \sum_{a=1}^{n} \Pr[h(x)=a,\; h(y)=a]$

   2. 哈希函数

      直接定址法：H(key) = a * key + b

      除留余数法：H(key) = key % p（小于等于表长的最大质数。不能选接近2或10的幂）

      双哈希：H(key, i) = (H~1~(key) + i * H~2~(key)) % m （i是尝试次数，从0开始。m是数组长度）

      全域哈希：H~ab~(key) = ((a * k + b) % p) % m

      p：一个大于键的全集大小U的质数（比如所有键都在0 - 2000，就可以选一个2003）

      a：在1到p-1里选一个整数

      b：在0到p-1里选一个整数

## 排序

以下均以升序为例：

1. 选择排序：每次找到整个未排序集中最小的元素，把它放到最前面

   1. 找到未排序集中最小的元素
   2. 将它与整个数组最左边的元素交换
   3. 现在最右边的一个元素是已排序集，剩下的是未排序集

   * 最好 / 最坏：O(n^2^)

     

2. 插入排序：每次将未排序集中的一个元素置入已排序集中的正确位置

   1. 将最左边的一个元素作为初始已排序集
   2. 检查已排序集是否是整集
   3. 将未排序集中最左边的元素放入已排序集中的正确位置

   * 最好：O(n)（完全正序）

   * 最坏：O(n^2^)（完全逆序）

     

3. 冒泡排序：每次比对一个元素与它右侧的元素大小，向左推进

   1. 检查已排序集是否为整集
   2. 将最左边的元素与其右侧元素作比较，如果顺序不对就交换
   3. 指针向右移动一位，继续第二步
   4. 每次遍历完一次未排序集，最右侧元素一定是最大或最小的，这就是已排序集

   * 最好：O(n)（完全正序）

   * 最坏：O(n^2^)（完全逆序）

     

4. 快速排序：递归地将整个集以基准元素分为一大一小两部分

   挖坑法：在数组中选择一个指针，比这个指针小的元素放到一边，比它大的元素放到另一边，这样就得到了指针和两个分类过的数组，接着再递归排序左右两个数组

   1. 维护数组开头和末尾两个指针
   2. 把第一个元素当作基准（随机挑选基准的话会造成额外的运算，效果没变化）
   3. 两个指针交替执行以下操作（以右侧指针、升序为例）
      1. 如果指针位置为基准，跳过回合
      2. 如果指针位置的元素小于另一个指针位置的元素（基准），将两个指针的元素交换位置
      3. 如果两个指针位置相同，表示排序完成

   * 最好：O(nlog n)（每个基准值都正好是子数组中位数）

   * 最坏：O(n^2^)（每个基准值都是当前范围内的最值）

     

5. 归并排序：递归地将整个集分为两个部分分别排序并合并

   合并的时候，在两个数组中分别维护两个指针，对两数组中相同位置的元素进行比较，再按顺序加入合并数组中

   * 最好：O(nlog n)

   * 最坏：O(nlog n)

     

## 复杂度

1. 渐进分析

   1. 有哪些基本运算

      * 基本数学运算
      * 变量赋值
      * 提取数组元素 / 获得对象（结构体）的属性
      * 调用函数
      * 返回值
      * 比较

      例如：在一个for(int i = 0, i < n, i++)循环中，int i = 0是一次运算，i < n总共会比较n+1次，i++会运算n次，最后整个循环运算2n + 2次

      

   2. 三种复杂度注释

      大O：运算的上界（不会比这个消耗更多了）
      $$
      T(N)=O(g(N))\quad \text{if}\quad\exist c,n_0\in R^+\quad\text{s.t.}\quad T(N)\le cg(N)\quad\text{when}\quad N\ge n_0
      $$

      * <big>**大O不是精确界**</big>，不过习惯上找最接近的（反过来说，只要比我增长快的函数都可以是我的大O  <small>考试有用</small>）
      * O(1) < O(log n) < O(n) < O(nlog n) < O(n^2^) < O(n^3^) < O(2^n^) < O(n!) < O(n^n^)
      * 大O表示会忽略掉原本的算式中的低阶项和常数项，**但在n次数较小时**，即便是大O复杂度较低的算法也可能不如一个复杂度高，然而常数项小的算法。

      

      大$\Omega$：运算的下界（不会比这个消耗更少了）
      $$
      T(N)=\Omega(g(N))\quad\text{if }\exist c,n_0\in R^+\quad\text{s.t.}\quad T(N)\ge cg(N)\text{ when } N\ge n_0
      $$
      大$\Theta$：运算的紧确界（上下界相同）
      $$
      T(N)=\Theta(g(N))\quad \text{当且仅当 } T(N)=\Omega(g(N))\text{ and }T(N)=\Omega(g(N))\quad
      $$

   

2. 相对性质

   1. 传播性

      $T(N)=\Theta (g(n))\text { and } g(n) = \Theta(h(n))\quad\Rightarrow  \quad T(N)=\Theta(h(n))$

   2. 自身性

      $T(N)=\Theta(T(N))$ （对O和$\Omega$也是）

   3. 对称性

      $T(N)=\Theta(g(n))\text{ 当且仅当 }g(n)=\Theta(T(N))$

   4. 转换对称性

      $T(N)=O(g(n))\text{ 当且仅当 }g(n)=\Omega(T(N))$

      

3. 大O的定理

   1. 如果g(n)是次数为d的非负多项式$g(n)=a_d\cdot n^d+a_{d-1}\cdot n^{d-1}+\cdots+a_2\cdot n^2+a_1\cdot n+a_0$

      则 g(n)为$O(n^d)$

   2. 相乘就相乘

      $g_1(n)=O(f_1(n)), g_2(n)=O(f_2(n))$

      则 $g_1(n)\cdot g_2(n)=O(f_1(n)\cdot f_2(n))$

   3. 相加取最大

      $g_1(n)=O(f_1(n)), g_2(n)=O(f_2(n))$

      则 $g_1(n)+g_2(n)=O(max(f_1(n),f_2(n)))$

   4. 如果$g(n)=(\log n)^a+n^b$且 a, b > 0 则 $g(n)=O(n^b)$（n比log n牛逼）

      如果$g(n)=(\log n)^a+b\cdot (\log n)^x$且 b > 0, a > x > 0 则 $g(n)=O((\log n)^a)$（指数大的更牛逼）

   5. 如果$g(n)=a^n+n^b$且 a > 1 则 $g(n)=O(a^n)$（n当指数更牛逼）

      如果$g(n)=a^n+b^n$且 a > b > 1 则 $g(n)=O(a^n)$（底数大的更牛逼）

      

4. 分析技巧

   1. 循环和嵌套

      ```c++
      for (int i=0; i<n; ++i) {             // 第一层：运行 n 次
          for (int j=n; j>0; j /= 2) {      // 第二层：j 每次除以 2，运行 log n 次
              for (int k=n; k>0; k /= 2) {  // 第三层：k 每次除以 2，运行 log n 次
                  sum++;                    // 最内层操作：O(1)
              }
          }
      }
      ```

      每一层的次数乘起来就可以得到总复杂度

   2. 拼接

      先执行一段代码再执行另一段代码，总复杂度就是它们的复杂度中的最大值（加起来）

   3. 条件语句

      总复杂度是if块内的复杂度和else块内的复杂度取大

## 二叉树的基础概念

1. 形式

   1. 满二叉树：所有非叶节点都有两个子节点
   2. 完全二叉树：一层一层从左往右排（除了最下层从左往右，其他层的节点数全部达到最大值。）
   3. 完美二叉树：三角形的树（完全填满）

   

2. 节点实现

   需要的属性：

   1. node.parent：父节点的地址
   2. node.leftchild：左子节点的地址
   3. node.rightchild：右子节点的地址
   4. node.element：该节点的值

   * 这里是用指针实现的

   

3. **遍历顺序**

   * 前序：本节点→左子树→右子树
   * 中序：左子树→本节点→右子树
   * 后序：左子树→右子树→本节点

   使用递归实现

   

4. **知序重建**

   已知一个树的前/后序和中序的遍历结果，可以重建出这个树：

   1. **前序遍历的第一个节点就是树的根节点**，以此可以将中序遍历的结果划分为“左子树-根节点-右子树”的三个部分。

   2. 重复这一操作即可得到重建结果

      

   * 后序遍历的最后一个节点是树的根节点

   

## 二叉搜索树

1. 概念

   对于任意一个节点都有：

   * 左子树的所有值都比这个节点小

   * 右子树的所有值都比这个节点大

     

     所有节点从小到大排序，在当前节点前面的值叫前驱，在后面的叫后继

   * 前驱：左子树的最大值

   * 后继：右子树的最小值

   

2. 基本操作

   1. 搜索

      最值：最小值是树最左侧的值，最大值是树最右侧的值

      任意值：二分搜索

      

   2. 插入

      比较插入值与当前节点，更大往左，更小往右，重复直到没有子节点

      

   3. 删除分情况

      1. 无子树：直接删
      
      2. 有一个子树：子树的根节点顶替被删节点的位置
      
      3. 有两个子树：把被删节点替换为它的后继，再把原本的后继删除
      
         
      
      * 连续删除后，容易向左失衡
      
      

## AVL树



1. 特征：

   * 是二叉搜索树

   * 每个节点有平衡参数：左子树高 - 右子树高

   * 平衡二叉树的高度随节点数n的增长数率为O(log n)

   

2. 平衡

   1. 判断失衡类型

      1. 看失衡节点的值，2为左失衡，-2为右失衡
      2. 左失衡看左子树，右失衡看右子树
      3. 再子树根节点的值，1为左型，-1为右型

   2. 不同类型怎么转

      1. LL型

         右旋失衡节点

      2. RR型

         左旋失衡节点

      3. LR型

         先左旋左子树根节点，再右旋失衡节点

      4. RL型

         先右旋右子树根节点，再左旋失衡节点

   3. 代码实现

      单次旋转

      ```java
      private AvlNode rotateRight(AvlNode k2) {
          AvlNode k1 = k2.left;
          // 转
          k2.left = k1.right; // 左子树的右子树变为失衡节点的左子树
          k1.right = k2; // 失衡节点变为左子树的右节点
          // 更新高度
          k2.height = Math.max(height(k2.left), height(k2.right)) + 1; // +1是自己
          k1.height = Math.max(height(k1.left), k2.height) + 1;
          return k1; // 返回新的根节点（调用是 t = rotateRight(t);的形式)
      }
      ```

      

3. 插入和删除

   插入和删除后都要检查节点是否失衡

   * 删除时可能出现多于一个节点失衡的情况，从下向上依次对失衡节点执行平衡操作

   

## 堆

1. 性质

   * 堆是一个完全二叉树

   最大 / 小堆：

   * 堆的任意父节点都严格大于 / 小于它的子节点
   * 根据递归性质，根节点一定会是整个堆的最大 / 小值

   

2. 建堆（以下全部以最大堆为例）

   对于任意完全二叉树，需要方法把它们调整成符合最大堆的性质

   实现逻辑：

   从第一个非叶节点开始，对这个节点执行下沉操作，下沉操作会让这个节点一直往下直到它的所有子节点都小于它或它不再有子节点

   

3. 堆排序

   把堆的根节点与末尾节点交换位置然后无视它，接着对新的根节点执行下沉操作

   其实就是：交换 → 下沉，如此循环直到结束

   

4. 代码实现

   1. 下沉

      ```c++
      // 传入参数：堆a，下沉目标节点下标k，末尾节点n
      void DownAdjust(int a[], int k, int n) {
          int i = k, j = i * 2; // j就是目标节点的左子节点
          while(j <= n) { // 检查i是否没有子节点
              if(j+1<=n && a[j+1]>a[j]) j++; // 让j对应的是i更大的子节点的下标
              if(a[i] < a[j]) { // 如果i的值更小，执行下沉
                  swap(a[i], a[j]);// 交换j和i的位置，然后让j成为新的i
                  i = j;
                  j = i * 2;
              }else{
                  break; // 如果i的值更大则i已经到正确位置，函数结束
              }
          }
      }
      ```

   2. 堆排序

      先建堆，再排序

      ```c++
      void HeapSort(int a[], int n) {
          // 建堆
          for(int i = n / 2; i >= 1; i--) { // 让i初始化为末尾节点的父节点（第一个非叶节点）
              DownAdjust(a, i, n);
          }
          // 排序
          for(int i = n; i >= 2; i--) { // 从末尾元素开始
              swap(a[1], a[i]);
              DownAdjust(a, 1, i - 1) // i - 1, 把已排序元素忽略
          }
      }
      ```



## 图

1. 概念

   图是一堆点和边的集合的集合

   G = (V, E)

   V = {A, B, C, D}

   E = {(A, B), (C, D)}

   * 对于有向图而言，<A, B> 和 <B, A> 并不是同一个东西

   度：一个点有几条弧就有几度；有向图还可以分为入度和出度

   连通：任意两点间都存在路径

   全联通：任意两点间都存在边

   

   有向图

   强连通：任意两点间都存在路径

   弱连通：无视方向才任意两点间存在路径

   全连通：任意两点间都有双向边

   DAG（有向无环图）：有向且无环，无连通性要求

   

2. 表示方法

   邻接矩阵：

   行是起点，列是终点，每个位置用0或1表示是否有连接，有权图的话就写权重

   无向图的邻接矩阵是对称的

   邻接表：

   一个数组或list，每个位置表示一个顶点，每个顶点维护一个链表，元素是这个顶点的邻居。有向表就只填出的邻居

   

   

3. 最小生成树

   1. Prim算法

      看顶点找MST

      ```c++
      // 核心逻辑：维护一个lowcost数组，记录每个点是否在当前树中，以及其他点到树的最小距离
      #define INF 2147483647
      #define MAXV 100
      int Prim(int G[MAXV][MAXV], int n , int v) { // 传入邻接矩阵，节点数n，起始节点v
          int lowcost[MAXV];
          int sum = 0;
          // 初始化lowcost数组，遍历每个节点
          for(int i = 0; i < n; i++) {
              if(i == v) lowcost[i] = 0; // 如果是起始节点，设置为0（已在树中）
              else lowcost[i] = G[i][v]; // 否则，设置为这个节点到起始节点的距离
          }
          // 找到距离当前树最近的节点，加入树，并更新lowcost数组
          for(int i = 1; i < n; i++) {
              int mindist = INF; // 用来记录当前找到的最小距离
              int k = -1; // 用来记录距离最近的节点下标
              for(int j = 0; j < n; j++) { // 遍历lowcost数组
                  if(lowcost[j]!=0 && lowcost[j]<mindist) { // 如果这个节点不在树中（不为0）且距离更近
                      mindist = lowcost[j]; // 更新最近距离
                      k = j; // 更新最近节点
                  }
              }
              if(mindist == INF) return -1; // 如果循环结束mindist还是INF，说明图不连通，直接返回
              lowcost[k] = 0; // 把找到的最近节点加入树中
              sum += mindist; // 累加边权
              // 更新lowcost：算其他点到新加入的点的距离会不会更小
              for(int j = 0, j < n; j++) {
                  if(lowcost[j]!=0 && G[j][k]<lowcost[j]) { // 如果不在树中，且到新点的距离更短就更新
                      lowcost[j] = G[j][k];
                  }
              }
          }
          return sum;
      }
      ```

      * 复杂度分析：

        最大的部分是一个嵌套for循环，所以复杂度就是O(n^2^)

      * 其他输出：

        如果想要直接返回得到的MST，可以开一个空邻接矩阵，每次加点进去的时候改为更新邻接矩阵的值

      

   2. Kruskal算法

      看边找MST

      ```c++
      struct Edge { // 这个算法只看边，所以定义一个边结构
          Vertex u, v;
          int w;
      };
      
      int Find(int p[], int x) { // 并查集的Find，会返回值x的根（标签）
          if(p[x] == x) return x;
          else return p[x] = Find(p, p[x]);
      }
      
      int Kruskal(Edge E[], int n, int m) {
          int p[MAXV];
          int sum = 0;
          int cnt = 0; // 用于数已经加入了几条边
          MergeSort(E, 0, m-1); // 把E数组里的边根据权重排序
          for(int i = 0; i < n; i++) { // 初始化并查集
              p[i] = i;
          }
          for(int i = 0; i < m; i++) { // 遍历每条边
              int pu = Find(p, E[i].u); // 获得这条边的两个定点的根（标签）
              int pv = Find(p, E[i].v); 
              if(pu != pv) { // 如果两个节点根不同（不在同一连通分量）
                  p[pv] = pu; // 把一个节点的根换成另一个的
                  sum += E[i].w; // 累加权重
                  
                  cnt++; // 加入一条边，如果已经有n-1条边了就直接跳出循环（已经全连通了）
                  if(cnt == n-1) break;
              }
          }
          if(cnt != n-1) return -1; // 如果枚举完边数不是n-1则图也不连通
          return sum;
      }
      ```

      复杂度分析：

      排序部分用的归并排序，是O(mlogm)，并查集初始化是O(n)

      由于是连通图，n个点至少要有n-1条边，所以m >= n-1，所以mlogm一般大于n

      所以复杂度是O(mlogm)（复杂度取决于排序效率）

   

4. 最短路径

   1. Dijkstra算法

      求某一个点到所有点的最小路径

      跟Prim算法很像，需要去维护一个记录起点到其他点的当前最小距离的数组

      ```c++
      #define MAXV 100
      #define INF 2147483647
      void Dijkstra(int G[MAXV][MAXV], int n, int v) { // 传入邻接矩阵，顶点数n，起点v
          int dist[MAXV]; // 维护当前从起点出发到其他点的最小距离
          bool flag[MAXV]; // 记录每个点是否已经确定最小距离
          int pre[MAXV]; // 记录每个点最短路径的前驱
          // 初始化dist，flag和pre
          for(int i = 0; i < n; i++) {
              dist[i] = G[v][i]; // 记录起点直接到每个点的距离
              flag[i] = 0; // 所有点最开始都没确定
              if(G[v][i]<INF) pre[i] = v; // 只要与起点相连，前驱就设为起点，否则设为无
              else pre[i] = -1;
          }
          flag[v] = 1; // 确定起点已经是最小距离
          dist[v] = 0;
          // 确定距离起点最近的点，更新其他点到起点的距离
          for(int i = 1; i < n; i++) { // 从1开始再找n-1个点
              int mindist = INF; // 记录当前找到的最小距离
              int k = -1; // 记录距离最近的点
              for(int j = 0; j < n; j++) { // 遍历所有点
                  if(flag[j]==0 && dist[j]<mindist) { // 如果还未被确定且距离最小 
                      mindist = dist[j];
                      k = j;
                  }
              }
              if(mindist == INF) break; // 如果起点无法到达任何其他的点就直接跑路
              flag[k] = 1; // 确认这个点已经是最小距离
              // 看看从起点出发经过这个点到其他点会不会更近
              for(int j = 0; j < n; j++) {
                  // 未确定，可到达，（起点到达刚确定的点的距离+刚确定点到这个点的距离）比记录的短
                  if(flag[j]==0 && G[k][j]<INF && dist[k]+G[k][j]<dist[j]) {
                      dist[j] = dist[k]+G[k][j]; // 更新最小距离
                      pre[j] = k; // 更新前驱
                  }
              }
          }
      }
      ```

      复杂度分析：嵌套for循环，所以是O(n^2^)

      

   2. Floyd算法

      计算任意点到任意点的最短路径和长度

      支持负边权，但是不能有负权环

      ```c++
      #define MAXV 100
      #define INF 2147483627
      
      void Floyd(int G[MAXV][MAXV], int n) {
          int D[MAXV][MAXV]; // 记录两个点间的最小路径长
          int Path[MAXV][MAXV]; // 记录任意路径的终点前驱
          for(int i = 0; i < n; i++) {
              for(int j = 0; j < n; j++) { // 遍历矩阵所有位置
                  D[i][j] = G[i][j]; // D矩阵初始跟邻接矩阵完全一样
                  if(i!=j && G[i][j]!=INF) Path[i][j] = i; // 只要不是自己到自己，或不能直达，就没前驱
                  else Path[i][j] = -1; // 没前驱就设为-1
              }
          }
          // 更新两个矩阵
          for(int k = 0, k < n; k++) { // 要把每个点都做中间点更新一次
              for(int i = 0; i < n; i++) {
                  for(int j = 0; j < n; j++) { // 遍历矩阵所有位置
                      // 起点到中间点和中间点到终点都要有路。如果经过中间点的路更短就更新
                      if(D[i][k]!=INF && D[k][j]!=INF && D[i][k]+D[k][j]<D[i][j]) {
                          D[i][j] = D[i][k]+D[k][j]; // 更新新的最短路径
                          Path[i][j] = Path[k][j]; // 更新前驱
                      }
                  }
              }
          }
      }
      ```

      复杂度分析：三层for循环嵌套，所以是O(n^3^)

      

   3. Bellman-Ford算法

      不像Dijkstra一样层层确定最短路径，通过写死的循环不断试探到达一个点会不会有更短的路径

      可以确定含负权的图的最短路径，可以检测负环（可以使最短路径无限变小的情况）

      ```c++
      #define MAXV 100
      #define INF 2147483647
      
      bool BellmanFord(int G[MAXV][MAXV], int n, int v) { // 邻接矩阵，顶点数，起点
          // 这里的dist和pre跟Dijkstra算法是一样的
          int dist[MAXV];
          int pre[MAXV];
          for(int i = 0; i < n; i++) {
              dist[i] = G[v][i];
              if(G[v][i] < INF) {
                  pre[i] = v;
              } else {
                  pre[i] = -1;
              }
          }
          dist[v] = 0;
          pre[v] = -1;
          // 执行n-1次松弛
          for(int k = 1; k < n; k++) {
              bool updated = false;
              for(int i = 0; i < n; i++) {
                  for(int j = 0; j < n; j++) { // 这两层循环其实就是遍历了邻接矩阵的所有位置（边）
                      // 如果i可达，且通过G[i][j]这条边到达j比先前记录的距离更近，就更新
                      if(dist[i]!=INF && G[i][j]!=INF && dist[i] + G[i][j] < dist[j]) {
                          dist[j] = dist[i] + G[i][j]; // 更新距离
                          pre[j] = i; // 更新前驱
                          updated = true;
                      }
                  }
              }
              if (!updated) break; // 如果这一轮没有更新，就表示已经完成了
          }
          // 确定负环
          for(int i = 0; i < n; i++) {
              for(int j = 0; j < n; j++) { // 遍历每条边
                  // 这里的条件跟上面松弛是一样的。如果循环结束了还可以继续松弛，就说明存在负环
                  if (dist[i]!=INF && G[i][j]!=INF && dist[i] + G[i][j] < dist[j]) {
                      return false; // 存在负环的话算法就无效了
                  }
              }
          }
          return true;
      }
      ```

   

   

## 哈希

1. 装填因子

   a = n / m

   n是元素总数，m是表长

   

2. 哈希函数

   直接定址法：H(key) = a * key + b

   除留余数法：H(key) = key % p（小于等于表长的最大质数。不能选接近2或10的幂）

   双哈希：H(key, i) = (H~1~(key) + i * H~2~(key)) % m （i是尝试次数，从0开始。m是数组长度）

   

3. 全域哈希

   构造一个哈希族，使满足：

   Pr[H(key~1~) = H(key~2~)] <= 1/m

   直观理解就是，从一个哈希族里随机抽一个哈希函数，两个键映射到同一个位置的概率应该小于 1/m

   一个经典构造方法：H~ab~(key) = ((a * k + b) % p) % m

   p：一个大于键的全集大小U的质数（比如所有键都在0 - 2000，就可以选一个2003）

   a：在1到p-1里选一个整数

   b：在0到p-1里选一个整数

   

4. 哈希签名

   1. 布尔矩阵

      每个文档会产生一个shingle集合。假设我们有两个文档：

      - 文档 S1 包含 shingle：{A, B, F, G}
      - 文档 S2 包含 shingle：{C, D, E}

      把所有出现过的shingle列成行，把文档列成列，有就填1，没有就填0，这就是**布尔矩阵**，也叫特征矩阵。

   2. 签名矩阵

      对于一个文档，取它布尔矩阵中为1的行（它的shingle)

      对这些行进行哈希映射，得到其中最小的哈希值

      签名矩阵每一行是一个哈希函数，每一列是一个文档。每一个元素就是这个文档用这个哈希函数后得到的最小哈希值。

      只需要得到两个文档在签名矩阵中相同的行数，除以总行数就可以得到这两个函数的Jaccard相似度了

      

      一些符号：

      Sig(S, i)：签名矩阵的一个格子。S是文档（列），i是哈希函数（行）。存的就是最小哈希值

      min~pi~ pi(C)：集合C在排列pi下的最下哈希值

      Pr(h(S1) = h(S2)) = Sim(S1, S2)：核心定理



## 更多排序

1. 希尔排序

   ```c++
   void ShellSort(int a[], int n) { // 传入数组和数组的长度
       for(int d = n/2; d >= 1; d >= 2) { // 分组数量以数组长度的一半开始，每次减半，直到为1
           for(int i = d; i < n; i++) { // 从组里第二个元素开始插入
               if(a[i] < a[i-d]) { // 如果比前一个元素大就向前挪
                   int temp = a[i];
                   int j = i-d; // 前一个元素为j
                   do {
                       a[j+d] = a[j]; // 把j移到i的位置
                       j -= d; // 去检查再前一个元素
                   }while(j>=0 && a[j]>temp); // 如果j<0了就说明前面已经没有元素了
                   a[j+d] = temp; // 把插入元素放回正确的位置
               }
           }
       }
   }
   ```

2. 基数排序

   ```c++
   #define MaxDigit 3
   #define Radix 10
   
   struct Node {
       int data;
       struct Node *next;
   };
   
   void RadixSort(Node *&p) {
       Node *h[Radix], *t[Radix]; // 创建基数个桶
       for(int i = 0; i < Radix; i++) {
           h[i] = t[i] = NULL;
       }
       int b = 1;
       for(int d = 1; d <= MaxDigit; d++) { // 一共循环最大位数轮
           while(p!=NULL) {
               int k = p->data/b%Radix; // 计算该放到哪个桶里
               if(h[k]==NULL) {
                   h[k] = t[k] = p;
               } else {
                   t[k]->next = p; t[k] = p
               }
               p = p->next;
           }
           b *= Radix;
           // 把桶里的取出来连成新链表
           for(int i = 0; i < Radix; i++) {
               if(h[i] != NULL) {
                   if(p==NULL) {
                       p=h[i];r=t[i];
                   } else {
                       r->next = h[i]; r=t[i];
                   }
                   h[i] = t[i] = NULL
               }
           }
           r->next = NULL; // 补全链表结构
       }
   }
   ```
