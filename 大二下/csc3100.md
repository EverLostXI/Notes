## 数组与Java基础

1. Java中的内存管理

   内存的结构：

   * static data， 代码，以及heap（堆）在内存的一侧；stack（栈）在内存的另一侧。
   * stack和heap的扩展方向是相对的
   * stack由电脑自动管理，heap由程序员手动管理

   Java可以自动进行垃圾回收，它会自动检查heap上没有被引用的数据并标记

   

2. 数组

   1. 特征：
      * 有序的（Ordered）：意思是里面元素的位置一旦创建就被确定了（不是Sorted）
      * 定长
      * 所有元素数据类型相同

   2. 声明与初始化

      ```java
      type[] identifier = new type[length]; // declare
      identifier[0] = xxx; // assigning value
      int[] identifier = {1, 2, 3}; // easy way to declare and initialize an array
      ```

      Java中的对象一般都是通过引用传入，引用是直接传对象的地址

      Java中每个数组都是对象，因此作为参数传入的时候传入的也是引用
   
   3. 多维数组
   
      ```java
      int[][] A = new int[2][3]; // declare a 2 dimensional array

      ```

	* Java中的变量声明，左边是变量类型，右边是分配空间

## 排序

以下均以升序为例：

1. 选择排序：每次找到整个未排序集中最小的元素，把它放到最前面

   1. 找到未排序集中最小的元素
   2. 将它与整个数组最左边的元素交换
   3. 现在最右边的一个元素是已排序集，剩下的是未排序集

   * 最好 / 最坏：O(n^2^)

     

2. 插入排序：每次将未排序集中的一个元素置入已排序集中的正确位置

   1. 将最左边的一个元素作为初始已排序集
   2. 检查已排序集是否是整集
   3. 将未排序集中最左边的元素放入已排序集中的正确位置

   * 最好：O(n)（完全正序）

   * 最坏：O(n^2^)（完全逆序）

     

3. 冒泡排序：每次比对一个元素与它右侧的元素大小，向左推进

   1. 检查已排序集是否为整集
   2. 将最左边的元素与其右侧元素作比较，如果顺序不对就交换
   3. 指针向右移动一位，继续第二步
   4. 每次遍历完一次未排序集，最右侧元素一定是最大或最小的，这就是已排序集

   * 最好：O(n)（完全正序）

   * 最坏：O(n^2^)（完全逆序）

     

4. 快速排序：递归地将整个集以基准元素分为一大一小两部分

   挖坑法：在数组中选择一个指针，比这个指针小的元素放到一边，比它大的元素放到另一边，这样就得到了指针和两个分类过的数组，接着再递归排序左右两个数组

   1. 维护数组开头和末尾两个指针
   2. 把第一个元素当作基准（随机挑选基准的话会造成额外的运算，效果没变化）
   3. 两个指针交替执行以下操作（以右侧指针、升序为例）
      1. 如果指针位置为基准，跳过回合
      2. 如果指针位置的元素小于另一个指针位置的元素（基准），将两个指针的元素交换位置
      3. 如果两个指针位置相同，表示排序完成

   * 最好：O(nlog n)（每个基准值都正好是子数组中位数）

   * 最坏：O(n^2^)（每个基准值都是当前范围内的最值）

     

5. 归并排序：递归地将整个集分为两个部分分别排序并合并

   合并的时候，在两个数组中分别维护两个指针，对两数组中相同位置的元素进行比较，再按顺序加入合并数组中

   * 最好：O(nlog n)

   * 最坏：O(nlog n)

     

## 复杂度

1. 渐进分析

   1. 有哪些基本运算

      * 基本数学运算
      * 变量赋值
      * 提取数组元素 / 获得对象（结构体）的属性
      * 调用函数
      * 返回值
      * 比较

      例如：在一个for(int i = 0, i < n, i++)循环中，int i = 0是一次运算，i < n总共会比较n+1次，i++会运算n次，最后整个循环运算2n + 2次

      

   2. 三种复杂度注释

      大O：运算的上界（不会比这个消耗更多了）
      $$
      T(N)=O(g(N))\quad \text{if}\quad\exist c,n_0\in R^+\quad\text{s.t.}\quad T(N)\le cg(N)\quad\text{when}\quad N\ge n_0
      $$

      * <big>**大O不是精确界**</big>，不过习惯上找最接近的（反过来说，只要比我增长快的函数都可以是我的大O  <small>考试有用</small>）
      * O(1) < O(log n) < O(n) < O(nlog n) < O(n^2^) < O(n^3^) < O(2^n^) < O(n!) < O(n^n^)
      * 大O表示会忽略掉原本的算式中的低阶项和常数项，**但在n次数较小时**，即便是大O复杂度较低的算法也可能不如一个复杂度高，然而常数项小的算法。
      
      
      
      大$\Omega$：运算的下界（不会比这个消耗更少了）
      $$
      T(N)=\Omega(g(N))\quad\text{if }\exist c,n_0\in R^+\quad\text{s.t.}\quad T(N)\ge cg(N)\text{ when } N\ge n_0
      $$
      大$\Theta$：运算的紧确界（上下界相同）
      $$
      T(N)=\Theta(g(N))\quad \text{当且仅当 } T(N)=\Omega(g(N))\text{ and }T(N)=\Omega(g(N))\quad
      $$

   

2. 相对性质

   1. 传播性

      $T(N)=\Theta (g(n))\text { and } g(n) = \Theta(h(n))\quad\Rightarrow  \quad T(N)=\Theta(h(n))$

   2. 自身性

      $T(N)=\Theta(T(N))$ （对O和$\Omega$也是）

   3. 对称性
   
      $T(N)=\Theta(g(n))\text{ 当且仅当 }g(n)=\Theta(T(N))$
   
   4. 转换对称性
   
      $T(N)=O(g(n))\text{ 当且仅当 }g(n)=\Omega(T(N))$
   
      
   
3. 大O的定理

   1. 如果g(n)是次数为d的非负多项式$g(n)=a_d\cdot n^d+a_{d-1}\cdot n^{d-1}+\cdots+a_2\cdot n^2+a_1\cdot n+a_0$

      则 g(n)为$O(n^d)$

   2. 相乘就相乘

      $g_1(n)=O(f_1(n)), g_2(n)=O(f_2(n))$

      则 $g_1(n)\cdot g_2(n)=O(f_1(n)\cdot f_2(n))$

   3. 相加取最大

      $g_1(n)=O(f_1(n)), g_2(n)=O(f_2(n))$

      则 $g_1(n)+g_2(n)=O(max(f_1(n),f_2(n)))$

   4. 如果$g(n)=(\log n)^a+n^b$且 a, b > 0 则 $g(n)=O(n^b)$（n比log n牛逼）

      如果$g(n)=(\log n)^a+b\cdot (\log n)^x$且 b > 0, a > x > 0 则 $g(n)=O((\log n)^a)$（指数大的更牛逼）

   5. 如果$g(n)=a^n+n^b$且 a > 1 则 $g(n)=O(a^n)$（n当指数更牛逼）

      如果$g(n)=a^n+b^n$且 a > b > 1 则 $g(n)=O(a^n)$（底数大的更牛逼）

      

4. 分析技巧

   1. 循环和嵌套

      ```c++
      for (int i=0; i<n; ++i) {             // 第一层：运行 n 次
          for (int j=n; j>0; j /= 2) {      // 第二层：j 每次除以 2，运行 log n 次
              for (int k=n; k>0; k /= 2) {  // 第三层：k 每次除以 2，运行 log n 次
                  sum++;                    // 最内层操作：O(1)
              }
          }
      }
      ```

      每一层的次数乘起来就可以得到总复杂度

   2. 拼接

      先执行一段代码再执行另一段代码，总复杂度就是它们的复杂度中的最大值（加起来）

   3. 条件语句

      总复杂度是if块内的复杂度和else块内的复杂度取大

      

## 递归的复杂度分析

1. 主定理

   如果整个算法的复杂度为形如：$T(n)\le a\cdot T(n/b)+O(n^d)$ 的式子

   则有：
   $$
   T(n) = \begin{cases} 
   O(n^d \log n) & \text{if } a = b^d \\ 
   O(n^d) & \text{if } a < b^d \\ 
   O(n^{\log_b(a)}) & \text{if } a > b^d 
   \end{cases}
   $$
   a：每次拆分出的子问题数量

   n / b：每个子问题的大小

   O(n^d^)：拆分这些问题所需要的固定步骤
